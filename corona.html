Corona - Visualization

Background

The above example shows a coordination visualization in D3 using the live properties Architecture. The data for the following visualization is gathered from Corona Example.

Interface

Different views are organized column wise

  • The first column from left consists of two scatterplot views.The Top Left scatterplot shows Tests Per Million drawn on horizontal axis vs Cases Per Million on vertical Axis. The Bottom Left scatterplot shows Deaths Per Million drawn on horizontal axis vs Tests Per Million on vertical Axis
  • The second column from left shows a Table view that shows all the data set. THe Continent coloumn is colored based on the categorical color legend assigned to continent.
  • The third column from left shows a Choropleth Map shows a Mercator projection of the world map. Each continent is colored based on the categorical color legend assigned to continent
  • The fourth column from left shows a Bubble chart view shows various bubbles drawn using size to represent the size of the circle
  • The fifth column from left shows a Parallel coordinate plot that is drawn using the same data. The color of the paths is drawn using categorical color legend assigned to attribute of the continent

All the views use the color encoding from color brewer 2.0 website to encode the color with the continent that the attribute belongs to.

Coordination

  • The two scatterplots on the first column from the left are coordinated by the range property. The Horizontal range of the top scatterplot is coordinated with the vertical range of bottom scatterplot.
  • The top scatterplot in the first column and the choropleth Map are coordinated in selection. Any selection made in either of the above will reflect in other.
  • The bottom scatterplot in the first column, table view and the Parallerl coordinate plot are coordinated in selection. Any selection made in the scatterplot view or table view will show the selected items in the three coordinated plots.
  • The scatterplots, table view, Choropleth map and bubble chart are coordinated in indication. Any item hovered in table view, Choropleth map and bubble chart will be highlighted in other plots.

Interaction

Scatter Plot

The current implementation supports either Zoom or brush on the scatterplot at a time. we decided to implement brush property instead. Brushing can be done in the scatterplot view by clicking the mouse anywhere on the scatterplot followed by drag and release to select a brushed region. Brusing in the above implementation is performed using rectangular brushed regions. Any items that fall in the brushed region are selected. Selected items are shown in the scatterplot using a black colored stroke.

Table View

selection in table view is done by double clicking on the row for selecting a single element. For selecting multiple rows click on a row and drag the mouse to the top or bottom based ont the direction in which the selection is to be made. There are performance issues associated with multiple row selection in case of table view in the current implementation which will be addressed soon which causes the multiple selection to not perform consistently always. The selected rows have a red colored text font. Indication is done in table view by hovering the mouse over the row. The indicated row is shown using a "teal" background color.

Choropleth Map

Selection is done on the choropleth map by clicking twice on a country with a small time gap between clicks otherwise the interaction will also trigger zoom along with selection. Selected regions are filled with grey color. Indication is done by hovering the mouse over the country. Indicated regions will be highlighted with a blue border and a reduced opacity.

Bubble Chart

Selection is done on the bubble chart by clicking twice on a bubble. Selected items are filled with grey color. Indication is done by hovering the mouse over the bubble. Indicated items will be highlighted with a white border and a reduced opacity.

Parallel Coordinate Plot

The above parallel coordinate plot does not allow selectioon or hovering. It will only display selected items when selected in a corresponding coordinated view.
