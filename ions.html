Ions - Visualization

Background

The above example shows a coordination visualization in D3 using the live properties Architecture. The data for the following visualization is gathered from Ions Example.

Interface

The visualization displays different spatial and temporal aspects of a simulated ion cloud trajectory organized column wise. A highly coordinated set of timeseries plots, table views and other views reveal different characteristics of oscillatory motion. The trajectory is displayed in a variety of ways by the views in the visualization:

  • The first column from left consists of three time series scatterplots that show ion motion over time relative to X, Y and Z axis.
  • The second, third, fourth and fifth columns consists of scatterplots drawn in a combination of different axis.
  • The third column and fourth column consists of a parallel coordinate plot thet shows the data in all the dimensions.
  • The sixth column from the left shows a table view with the actual {T,X,Y,Z} positions of points along the trajectory.
  • The sixth column from the left below the table view shows a Perpectiual view with time mapped as a color gradient.

All views map time into a translucent red-to-black color gradient that suggests (but does not map precisely to) decreasing energy of the ion cloud over time.

Coordination

  • The three scatter plots in the first column and the table view are coordinated in selection.
  • The three scatter plots in the first column are connected in range along horizontal axis.
  • The scatterplots in the second and third column from left consists of overview scatterplots for the detailed views in the scatterplot in fourth and fifth columns. The Top scatterplot in the second column and the top scatterplot in the fith column are overview-Detail pairs. The bottom scatterplot in the second column and the bottom scatterplot in the fourth column are overview and detail pairs.The bottom scatterplot in the third column and the bottom scatterplot in the fifth column are overview and detail pairs.
  • The parallel coordinate plot and the perpectual slider are coordinated in selection.

Interaction

Scatter Plot

The current implementation supports either zoom or brush on the scatterplot at a time. we decided to implement brush property instead. Brushing can be done in the scatterplot view by clicking the mouse anywhere on the scatterplot followed by drag and release to select a brushed region. Brusing in the above implementation is performed using rectangular brushed regions. Any items that fall in the brushed region are selected. Selected items are shown in the scatterplot using a black colored stroke. Brushing on the scatter plots in the first column will select items in the coordinated views to the plot on which the brushing is performed. Brushing performed on any of the overview plot will draw a detailed plot for the brushed region. Brushing performed on the detailed view will not have any effect as no coordination is defined for the brushing action.

Parallel Coordinate Plot

The above parallel coordinate plot does not allow selectioon or hovering. It will only display selected items from a corresponding corresponding coordinated view. The selected items are drawn using thick lines.

Table View

selection in table view is done by double clicking on the row for selecting a single element. For selecting multiple rows click on a row and drag the mouse to the top or bottom based ont the direction in which the selection is to be made. There are performance issues associated with multiple row selection in case of table view in the current implementation which will be addressed soon which causes the multiple selection to not perform consistently always. The selected rows have a red colored text font. The table view and the scatter in the first column are coordinated in selection.

Perpectual Slider

The perpectual slider and the parallel coordinate plot are coordinated in selection. Selection is performed in the perpectual slider by brushing. The slider is filled with a color gradient corresponding to the data. any selection performed in the slider will select the corresponding datapoints in the linked parallel coordinate plot.
